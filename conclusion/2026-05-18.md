## Session 1：检索问题排查与修复

### 1.1 创建通用评测问题集

- 创建 `tests/eval_questions.md`（98 题，5 大类）
  - 精准查询词 30 题 / 多文档综合 25 题 / 语义近似 20 组 / 隐含关联 15 题 / 边界测试 8 题
  - 覆盖 11 个分类，中英文混合比例 ~4:3:3

### 1.2 发现 Mify 检索返回空结果

- 测试 `search("clock gating")` 等多个查询，全部返回"未找到相关内容"
- 调用原始 API `/api/v1/datasets/{id}/retrieve`，确认 HTTP 200 但 `records: []`
- 同时 `/api/v1/datasets/{id}/documents` 返回 93 篇文档 → 文档存在，检索层有问题

### 1.3 根因定位

通过逐层排查发现：

| 排查项 | 结果 |
|--------|------|
| API 认证 | Bearer token 格式正确 |
| 文档端点 | 93 篇文档正常返回 |
| Retrieve 端点 | 返回空 records |
| payload 格式 | **缺少 `retrieval_model` 字段** |

**根因**：Mify 的 retrieve API 要求 payload 中包含 `retrieval_model` 配置（含 `score_threshold_enabled`），之前只传了 `query` 和 `top_k`，导致 API 静默返回空结果。

验证 payload 中加上以下字段后检索恢复：
```python
"retrieval_model": {
    "search_method": "hybrid_search",
    "reranking_enable": True,
    "reranking_model": {
        "reranking_provider_name": "langgenius/tongyi/tongyi",
        "reranking_model_name": "gte-rerank-v2_mi_sys",
    },
    "score_threshold_enabled": False,
    "weights": {
        "weight_type": "customized",
        "keyword_setting": {"keyword_weights": {}},
        "vector_setting": {
            "embedding_provider_name": "langgenius/azure_openai/azure_openai",
            "embedding_model_name": "text-embedding-3-small_mi_sys",
        },
    },
}
```

### 1.4 修复代码

- 修改 `core/mify_client.py` 的 `_fetch_one_variant()` 函数
- 在请求 payload 中补全 `retrieval_model` 配置
